# Portfolio Deliverables

This project is not done when the local app merely works. It is done when it can prove Agent PM competence in an interview or portfolio review.

## Required Deliverables

### 1. Runnable Local Demo

The reviewer should be able to run the app locally and see the core loop:

`task space -> eval draft generation -> human approval -> manual SBS collection -> review -> downloadable report`

Minimum demo data can be synthetic, but it must look like a real task-space evaluation, not toy lorem ipsum.

### 2. Sample SBS Report

The project should include one polished sample report generated by the app.

The report should demonstrate:

- task-space framing;
- why Doubao is the baseline;
- generated and approved eval cases;
- side-by-side outputs;
- evidence level per side;
- manual winner/unclear/tie notes;
- failure mode observations;
- product implications and next-step recommendations.

The report does not need final grader scores until the evaluation methodology session is complete.

### 3. README Story

The README should explain:

- the product thesis;
- why task-space SBS matters;
- why eval set generation is the key PM capability;
- what the MVP does and does not do;
- how to run the app;
- what the sample report proves;
- roadmap: grader, browser automation, coding-agent arena.

### 4. Interview Story

The project should support this narrative:

> I used my recommendation-strategy PM background to build a task-space SBS eval harness for AI products. The key product insight is that an agent product only matters if it beats strong product baselines in a concrete task space. I built the first loop around AI-assisted eval set generation, human approval, black-box output collection, and reportable failure analysis.

### 5. Resume Bullet

Draft:

> Built a local Agent Product SBS Eval Harness for comparing challenger AI products against strong baselines such as Doubao/Codex within a defined task space. Designed an AI-assisted eval-set generation workflow, human approval gate, manual black-box collection flow, evidence-level tracking, and downloadable SBS report to convert eval results into product improvement recommendations.

## Portfolio Quality Bar

- The app should feel like a PM workbench, not a generic form demo.
- The sample report should make a product judgment traceable.
- The generated eval cases should show task-space insight.
- The README should connect engineering choices to Agent PM judgment.
